Spec de Cascateamento — sales-spark core → 4 SaaS

Data: 2026-06-09

Premissa: o core (schema + edges + frontend + crons) está provado no NexvyOficinas. Cascatear = replicar o core idêntico + personalizar cada SaaS.

Alvos: NexvyBeauty, BarbeiroPro, NexvyFoods, NexvyGYM (cascas hoje).

Você está certo — com 2 ressalvas

Sim: o trabalho pesado foi feito uma vez no Oficinas. O resto é replicação mecânica + personalização. Duas ressalvas que o spec trata:

  1. Cada casca tem um vertical-template diferente no banco (Beauty=agendamentos/pacotes, Foods=cardapio/entregas, GYM=alunos/checkins, Barbeiro=booking). No reset (DROP SCHEMA) somem e viram o schema sales-spark — precisa ser intencional (greenfield, como no Oficinas).
  2. Domínios + branding + secrets + super-admin são por-app — domínios ainda são placeholders nos templates Traefik. São as entradas que você fornece.

3. Decisão: vertical de nicho — agora ou depois?

O sales-spark é CRM/atendimento genérico que serve qualquer nicho.

EstratégiaEntregaEsforço/SaaS
A — Core puro primeiro ⭐sales-spark completo + branding do nicho. 100% das funções, operante.~0,5 sprint
B — Core + vertical+ tabelas/UI específicas (Beauty: serviços; Foods: cardápio/delivery; GYM: treinos; Barbeiro: agenda cortes)+1-2 sprints

Recomendação: A primeiro (4 SaaS operantes rápido), B depois por demanda. Mesma lógica do Oficinas.

4. Runbook parametrizado

APP_DIR=NexvyBeauty

DOMAIN=nexvybeauty.com.br

Fase A — Banco (~10 min): link → extensions (vector/pg_trgm em public) → DROP SCHEMA → baseline-schema → grants → seeds → storage → realtime → cron.

Fase B — Edges (~8 min): cp functions → config.toml com ref → functions deploy.

Fase C — Frontend (~5 min): cp src → .env do project → branding (platform_settings) → npm build.

Fase D — Deploy + Auth: DNS → traefik template → deploy-vps.sh → secrets → Auth Site URL → super admin signup.

Automação proposta: empacotar Fases A-C num infra/cascade-core.sh <APP_DIR> <REF> <CONTAINER> <DOMAIN>. Fase D fica semi-manual (inputs seus).

7. Riscos & validação

  • Reset apaga vertical-template da casca → confirmar greenfield de cada.
  • DNS não propagado → Traefik não roteia. Validar curl https://$DOMAIN.
  • Secrets ausentes → IA/email inertes (não bloqueia subir).
  • Done por SaaS: curl HTTP 200 + login super admin + 161 tabelas no banco.

Conclusão

Cascateamento = 80% replicação mecânica (core idêntico, runbook provado) + 20% personalização (domínio/branding/secrets/super-admin). Estratégia A → 4 SaaS operantes em ~1,5 sprint. Vertical de nicho é incremento posterior.
