1Stack + a decisão-chave do porte

LP = TanStack Start (react-start + react-router + Nitro), React 19, Tailwind v4, Vite 8. Destino (apps/NexvyBeauty) = Vite + React-Router, React 18, Tailwind v3 → mismatch de framework.

Mas o porte é simples porque a LP não depende do framework:
  • Conteúdo todo em 1 arquivo: src/routes/index.tsx (JSX cru + hooks padrão + React.cloneElement — compatível com React 18).
  • Design todo em src/lp.css (classes próprias). NÃO usa shadcn/radix/Tailwind.
  • Sem loaders/server functions/SSR-dependente — só animações client-side.
  • Backend/DB/secrets/conexões: NENHUM (list_connections vazio). LP 100% estática.
Estratégia: extrair index.tsx como componente de página, trazer lp.css, dropar o createFileRoute, plugar como rota. Sem migrar backend.
