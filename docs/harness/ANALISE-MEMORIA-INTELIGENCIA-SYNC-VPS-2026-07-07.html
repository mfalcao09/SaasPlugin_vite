As 5 alavancas (ordem de retorno)

  1. Write-path único (mem-write) — um comando roteia (tipo, tema, conteúdo) → memo + linha no INDEX + tag.sh + push cérebro. Um funil = zero drift; o sync herda a disciplina.
  2. Git em ~/.claude (subset) — memory/hooks/skills/workstreams; commit automático no Stop. Histórico + rollback + substrato do sync da Q2. Excludes: segredos, caches, JSONL.
  3. Evals = o próximo degrau. ~20 golden queries no eval_gate.py (já existe no reindex 05UTC) · miner das 04:00 gera casos · métrica no Telegram. Regra: camada nova só entra com eval provando gap.
  4. Ciclo de ferramentas (o método que criou o v3): fricção → hook mínimo → medir 1 semana → promover/matar. Candidatos: PreCompact checkpoint (sessões pela metade!), SessionEnd distiller (lições automáticas), gate completed-com-prova.
  5. Higiene: frontmatter verificado_em:; job de frescor/expiry; aposentar números ilusórios.

Q2 — Sync Mac→VPS: análise e decisão

Eu discordo de continuar os Sprints 3–7 como estão porque o pipeline essencial já funciona e cada sprint novo cria outro sistema que exige o MESMO sync que te cansa. O que eu faria: fechar o perímetro — eleger SoT, endurecer o canal vivo, matar/adiar o resto. O risco de continuar: mais camadas `proposed` = mais frentes pela metade = mais fadiga, sem recall melhor.
