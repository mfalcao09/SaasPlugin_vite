Objetivo: BDR autônomo que pega lead congelado (Maps/IG/TikTok) → opt-in → handoff Duda. Auto-melhorável, sem queimar canal.

0. TL;DR — 3 vereditos + o keystone

LLM

Não há LLM local (ollama só embeddings). Você tem o gateway LiteLLM por tag: volume pro alto-volume, reasoning pra mensagem que vende. Medir PT-BR antes de fixar barato.

Auto-improver = Hermes agent

Corrigido: não é o openclaw (esse é só gateway na KVM4). O Hermes (NousResearch, 211k★) — não deployado ainda — auto-edita as próprias skills SEM eval-gate nativo. O medo de "agente que se reescreve" aplica-se → deploy pós-P1, com skill-gate no golden set.

Deliverability

API oficial + template não blinda de ban (Meta bane por quality-rating + exige opt-in). Frio quem toca é o anúncio, não o WhatsApp.

Keystone: fechar o loop de eval de conversão. Ele é (a) o portão de segurança da autonomia 24/7 e (b) o sinal de recompensa que faz a reingestão da cerebro-infra melhorar conversão, não só indexação. Sem ele, "24/7 autônomo + auto-melhora" = drift confiante rumo ao banimento.

2.3 Ordem de canal (deliverability)

  1. Ads (custom audience) — retargeta a lista raspada. Não queima número. Escala.
  2. Lead magnet ("escaneio grátis quantas clientes sumiram") — opt-in legítimo.
  3. WhatsApp API + template — só depois do opt-in. Protege quem optou; não autoriza frio.
  4. IG DM / WhatsApp frio — só contas descartáveis + circuit-breaker; alto risco de ban.

4. Auto-melhoria SEGURA

Você já tem (seguro por design): cerebro-infra/reindex reingere a memória diariamente (cron 05:00 UTC) e só promove o índice se passar do RAG_EVAL_THRESHOLD. Melhora o conhecimento sem mutar o comportamento. ✅

O que falta (keystone): o eval de hoje mede qualidade de retrieval, não conversão. Para "cada dia melhor em VENDER":

  1. Golden set de conversão — 50–100 conversas reais rotuladas (rota correta? falso-descarte? virou opt-in/demo?).
  2. Métrica de recompensa = opt-in / reply-sem-report / demo-booked — não reply cru.
  3. Alimentar sources/ com mensagens vencedoras → reindex diário absorve → auto-melhoria real e gated.
  4. Gate de deploy: nenhuma variante de prompt nem skill do Hermes (auto-editável, sem gate nativo) vai a produção sem passar (falso-descarte=0, opt-in≥baseline, report↓). Skills versionadas: mutação → candidata → eval promove.
É assim que "cada dia melhor" vira verdade sem virar "cada dia mais perto do ban".

7. Riscos / anti-padrões

  • Frio no WhatsApp (mesmo com template) → quality despenca → ban. Frio = ads.
  • Envio autônomo sem circuit-breaker → ban em escala. Gated no envio.
  • Scraping IG/TikTok viola ToS → contas descartáveis + throttle; Maps é o seguro.
  • Auto-melhoria medindo reply cru → agente vira agressivo → denúncia. Medir opt-in-sem-report.
  • Barão de token na 1ª mensagem → 1 msg ruim custa um número. reasoning na mensagem.
  • VPS única sem fallback → 24/7 vira "até saturar".
  • LGPD (Seção 11): base legal, opt-out em todo toque.
